                                1.1 算法性能度量

                                算法性能通过 时间复杂度 和 空间复杂度 两个方面进行度量，度量方法分为 事前分析法 和 事后统计法，不过常用前者来度量。

                                1.1.1 空间复杂度

                                算法的空间复杂度 S(n) 定义为该算法所耗费的存储空间，它是问题规模 n 的函数，记为：

                                S(n)=O(g(n))

                                一个程序在执行时除需要存储空间来存放本身所用的指令、常数、变量和输入数据外，还需要一些对数据进新房操作的工作单位和存储一些为实现计算多需要信息的辅助空间。

                                1.1.2 时间复杂度

                                一个语句的 频度 是指该语句在算法中被重复执行的次数。算法中所有语句的频度之和记为 T(n) ，它是该算法问题规模 n 的函数，时间复杂度主要分析 T(n) 的数量级 ，因此通常采用算法中基本运算的频度 f(n) 来分析算法的时间复杂度，记为：

                                T(n)=O(f(n))

                                其中，O 是指 T(n) 的数量级。算法的时间复杂度不仅依赖于问题的规模 n ，也取决于待输入数据的性质。

                                最好（下界）、最坏（上界）和平均（渐进）情况分别对应符号 Ω、O、Θ ，关系如图：

                                T(n) 取问题规模的频度 f(n) 中增长最快的项，且其系数置1，如：

                                f(n)=an3+b2+cn+d

                                则 T(n)=O(n3)

                                1. 加法规则：T(n)=T1(n)+T2(n)=O(f(n))+O(g(n))=O(max{f(n),g(n)})

                                2. 乘法规则：T(n)=T1(n)×T2(n)=O(f(n))×O(g(n))=O(f(n)×g(n))

                                3. 大小关系：O(1)<O(log2n)<O(n)<O(nlog2n)<O(n2)<O(n3)<O(2n)<O(n!)<O(nn)

                                1.2 算法复杂度分析

                                常见的算法复杂度的规模为：

                                O(1)<O(log2n)<O(n)<O(nlog2n)<O(n2)<O(n3)<O(2n)<O(n!)<O(nn)

                                算法的时间复杂度是通过求解最大规模 n 下对应的代码循环执行 t 轮的结果，即求解 t 与 n 之间的关系，算法的时间复杂度与 t 直接相关。

                                1.2.1 技巧

                                1.只关注循环执行次数最多的一段代码；

                                大O这种复杂度表示方法只是关注代码执行时间随数据规模增长的变化趋势。通常会忽略掉低阶、常量、系数，只需要记录一个最大的量级就可以。

                                2.加法法则：总复杂度等于量级最大的那段代码的复杂度；

                                3.乘法法则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积。

                                3.前中后缀表达式转换

                                一般我们常用的中缀表达式，中缀表达式不仅依靠运算符的优先级，也要处理括号的优先级；后缀表达式中没有括号，只有操作数和运算符，且运算符放在操作数的后面；前缀表达式也是一种没有括号的算术表达式，其运算符写在前面，操作数写在后面。

                                • 中缀表达式：a+b-c*d

                                • 前缀表达式：-+ab*cd（波兰式）

                                • 后缀表达式：ab+cd*-（与逆波兰式RPN一致）

                                转换方法：

                                (1) 直接法，利用计算的优先级将每一部分加上括号，然后将括号内的符合外移即可（如果是前缀表达式则前移，若为后缀表达式则后移），例如：

                                (a+b)*c+d-(e+g)*h对应的前缀表达式是-(+(*(+(ab)c)d)*(+(eg)h))得到-+*+abcd*+egh 。

                                2.遍历二叉树法，根据计算子式的优先级自底向上构建表达二叉树，然后根据先序遍历及后序遍历则可得到相应的表达式。

                                3.入栈法，中缀转后缀规则如下:

                                1. 从左到右扫描中缀表达式；

                                2. 遇到操作数直接将其写出；

                                3. 遇到操作符则将其入栈, 首先令其与栈顶运算符比较优先级，若优先级小于等于, 则栈顶出栈，该运算符入栈；一直循环, 直到该运算符大于栈顶优先级才入栈。左括号直接入栈，其他非右括号遇到左括号则直接入栈；遇到右括号，则一直出栈，直至匹配到第一个左括号为止。

                                4. 栈底直接入栈, 栈顶全部出栈，左括号相当于部分栈底, 右括号相当于部分栈顶, 均直接入栈出栈。

                                例：(a+b)*c+d-(e+g)*h-> ab+c*d+eg+h*-

                                如果是中缀表达式转前缀表达式则是：

                                1. 准备两个栈S1和S2；

                                2. 从右往左扫描表达式。若为数字则直接入栈S1；若为左括号则入栈S2；若为运算符，则与栈顶元素进行比较，若栈顶为左括号或大于等于栈顶运算符的优先级则直接入栈S2，若小于栈顶元素的优先级，则栈顶运算符出栈，该符号入栈。若为右括号，则持续出栈，直至遇到第一个左括号为止。

                                3. 循环步骤2，直至表达式扫描完成，然后出栈S1得到的序列便是前缀表达式。

                                 

                                4.表达式求值

                                一般采用逆波兰式（RPN）即后缀表达式求值，例如：(1+2)*3+4-(5+6)*7对应的值为-64 。

                                1. 准备操作数栈S；

                                2. 扫描RPN，遇到数字则直接入栈，遇到操作符，则从栈顶一次弹出2个操作数，将计算后的结果压入栈顶，直至RPN扫描结束即可得到最终的值。

                                例如：(1+2)*3+4-(5+6)*7对应的后缀表达式为 12+3*4+56+7*-

                                当然也可以用表达式二叉树进行运算，自底向上计算后合并结点得到最终的值。
